# Daily Notes — 2026-02-16 (Technical Integrity Scan)

## Purpose
Preliminary scan for:
- dict/float violations
- callback inconsistencies (CalmPressure-style)
- contradictions between organs
- blockers for ThoughtOrgan → MathBlockViewer integration

## Findings

### 1. Dict/Float Violations
- No direct violations detected yet, but ThoughtOrgan’s output format is not guaranteed to be a pure dict or pure float. 
- MathBlockViewer expects a stable math_block structure; ThoughtOrgan currently emits mixed packets depending on step() behavior.
- Recommendation: enforce a canonical math_block dict with float fields inside ThoughtOrgan before routing to viewer.

### 2. Callback Inconsistencies (CalmPressure Pattern)
- CalmPressure uses a callback-style pressure update that other organs do not follow.
- STMOrgan and HeartOrgan do not expose callback hooks; this may cause inconsistent packet flow.
- Recommendation: standardize a `post_step(packet)` callback across all organs.

### 3. Organ Contradictions
- No explicit contradictions detected, but:
  - HeartOrgan emits emotional polarity fields.
  - CalmPressure emits pressure deltas.
  - ThoughtOrgan emits semantic clusters.
  - WorldOrgan emits environmental context.
- These packets do not share a unified schema.
- Recommendation: define a shared “organ_packet” schema before integration.

### 4. Thought → MathBlockViewer Integration Blockers
- MathBlockViewer requires:
  - `float_core`
  - `dict_packet`
  - `metadata`
  - `resonance`
  - `pressure`
- ThoughtOrgan does not yet emit these fields consistently.
- Recommendation: extend ThoughtOrgan.step() to emit a full math_block structure.

## Summary
System is stable. No fatal contradictions. Integration requires schema alignment, not rewrites.

# MEMO COMMAND SET — MASTER LIST

## Organ Inspection Commands
memo thought          → run ThoughtOrgan.step() and show output
memo stm              → run STMOrgan.step()
memo calm             → run CalmPressure.step()
memo heart            → run HeartOrgan.step()
memo world            → run WorldOrgan.step()

## Math Block Viewer Commands
memo math <organ>     → show math block for the given organ
memo mb <organ>       → alias for math block viewer

## Code Inspection Commands
memo code <organ>     → print organ source code
memo code viewer      → print MathBlockViewer code
memo code rememger    → print rememger.py
memo code all         → print all organ code in sequence

## Memory / Notes Commands
memo today            → open today’s daily note
memo note <subject>   → create a subject-tagged note
memo recall <subject> → list notes by subject
memo recall all       → list all notes

## Snapshot / Restore Commands
memo snapshot         → create a snapshot
memo restore <name>   → restore a snapshot
memo history restore  → show restore_index.json lineage
memo history notes    → show notes_index.json metadata
