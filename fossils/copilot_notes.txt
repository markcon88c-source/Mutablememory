DESIGN PRINCIPLES GUIDING THE CREATURE
---------------------------------------------------------

1. Stability First
   Every build must preserve the creature’s ability to run.
   No organ may break another organ’s interface.

2. Fossilization as Memory
   Every failure is a developmental milestone.
   Fossils are not mistakes — they are lineage anchors.

3. Soft Integration
   New organs must be introduced gently, with soft calls,
   safe wrappers, and non-breaking defaults.

4. Unified Packet Philosophy
   All organs should eventually speak a shared symbolic
   packet, but early builds must remain permissive.

5. Human–AI Hybrid Memory
   The creature’s memory is distributed:
   - human notes
   - AI notes
   - auto-fossils
   - manual fossils
   - build notes
   - daily notes

6. Ritual Over Interface
   Commands like `mythic` and `mythos` are not utilities.
   They are rituals that stabilize the developmental loop.
